### Q4 : Le catalogue système - Table `pg_constraint`

La table **`pg_constraint`** 📜 appartient au schéma **`pg_catalog`**. [cite_start]Contrairement aux vues du schéma d'informations qui sont des standards SQL universels, cette table est spécifique au moteur interne de PostgreSQL[cite: 73, 76].

Voici les points clés à retenir sur son contenu :

* [cite_start]**Rôle** : Elle stocke la définition brute de toutes les contraintes de la base de données : clés primaires, étrangères, d'unicité, de validation (CHECK) et d'exclusion[cite: 76].
* [cite_start]**Structure technique** : Elle utilise des identifiants internes nommés **OID** (Object Identifiers) pour lier les contraintes aux objets (tables, colonnes)[cite: 115].
* **Types de contraintes (`contype`)** : Les types sont codés par un seul caractère dans la table :
    | Code | Signification |
    | :--- | :--- |
    | `p`  | Clé primaire (Primary Key) |
    | `f`  | Clé étrangère (Foreign Key) |
    | `c`  | Validation (Check) |
    | `u`  | Unicité (Unique) |

* **Colonnes clés** :
    * `conname` : Le nom de la contrainte.
    * `conrelid` : L'OID de la table possédant cette contrainte.
    * `confrelid` : L'OID de la table de référence (pour les clés étrangères uniquement).
