
* **La vue `table_constraints` :** Elle sert à lister l'ensemble des contraintes définies sur les tables d'une base de données. Son contenu inclut le nom de la contrainte, la table sur laquelle elle s'applique, et le type de contrainte (PRIMARY KEY, FOREIGN KEY, CHECK, UNIQUE, etc.).
* **Différence entre `constraint_column_usage` et `constraint_table_usage` :** * `constraint_column_usage` détaille les **colonnes** spécifiques qui sont concernées par une contrainte. 
    * `constraint_table_usage` liste les **tables** qui sont référencées par une contrainte. C'est particulièrement utile pour identifier les dépendances entre les tables créées par les clés étrangères.
* **La vue `key_column_usage` :** Elle sert à lister uniquement les colonnes qui sont impliquées dans des contraintes de type "clé", c'est-à-dire les clés primaires (PK), les clés étrangères (FK) et les contraintes d'unicité (UNIQUE).
